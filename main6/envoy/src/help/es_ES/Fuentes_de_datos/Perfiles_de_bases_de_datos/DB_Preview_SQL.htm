Instrucción SQL de vista previa de la base de datos

Sólo podrá ver estas pantallas si cuenta con integración de bases de datos.

 

Resumen:

Optativo.  Escriba las instrucciones SQL necesarias para volver a introducir los datos correspondientes a las columnas indicadas de una fila traducible con el fin de generar la vista previa.

Reglas de vista previa de la base de datos:

Habilita las reglas para generar la vista previa en contexto de los registros de la base de datos.

Conexión a la base de datos de vista previa:

Seleccione la conexión a la base de datos que debe utilizarse para grabar los datos de generación de la vista previa.

Instrucción SQL de inserción de vista previa:

Si no existe ningún registro con esa clave primaria, "Instrucción SQL de inserción de vista previa" inserta un nuevo registro en la base de datos especificada para la vista previa. Si el registro ya existe, se utiliza Instrucción SQL de actualización de vista previa. Esta instrucción SQL debe incluir todas las columnas necesarias para definir el registro. Utilice %%1%% para hacer referencia al primer elemento de la clave primaria y %%2%%, %%3%%, etcétera para hacer referencia a los elementos adicionales de una clave compuesta (vea también Tabla de registros pendientes). Utilice %%tablename.column%% para hacer referencia a los datos necesarios para cada columna del registro. Los nombres de la tabla y la columna deben adquirirse como parte de la instrucción SQL de lectura y deben definirse en la información de la columna (vea Columnas).

 

Ejemplo:

INSERT INTO items(short_desc, long_desc, dept, prod_id, properties) VALUES ('%%items.short_desc%%','%%items.long_desc%%',

'%%2%%','%%1%%', '%%items.properties%%')

Instrucción SQL de actualización de vista previa:

Si ya existe un registro con esa clave primaria, "Instrucción SQL de actualización de vista previa" actualiza un registro ya existente en la base de datos especificada para la vista previa. Si el registro no existe, se utiliza "Instrucción SQL de inserción de vista previa". Esta instrucción SQL se crea de la misma forma que la "Instrucción SQL de inserción de vista previa" antes descrita utilizando el comando Update en lugar de Insert, pero además se debe especificar una cláusula Where para asegurar que se actualice el registro correcto.

 

Ejemplo:

UPDATE items

SET short_desc='items.short_desc',

long_desc='items.long_desc',

properties='%%items.properties%%'

WHERE dept='%%2%%' AND prod_id='%%1%%'
