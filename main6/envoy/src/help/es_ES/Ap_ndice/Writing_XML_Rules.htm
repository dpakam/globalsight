Escribir reglas XML

El archivo de reglas XML define las reglas de acuerdo con las cuales System4 lee un archivo XML con un determinado esquema.  I

 

El archivo de reglas del extractor de XML es un documento XML. El mismo archivo de reglas puede incluir las reglas para varios tipos de documentos incorporando varios conjuntos de reglas (ruleset). Si la regla no define parte de la regla de extracción de un elemento o atributo, se extraerán de acuerdo con la regla predeterminada. Un archivo de reglas consta de cuatro tipos de elementos.

 

Especificación del archivo de reglas

Elemento <schemarules>

Éste es el elemento raíz del documento. Contiene uno o varios elementos <ruleset>.

Elemento <ruleset>

El <ruleset> define las reglas de un tipo de documento.

 

Atributo

Descripción

Obligatorio

schema

Éste es el nombre del elemento raíz del documento. Tenga en cuenta que el nombre es sensible a mayúsculas y minúsculas.

Sí

 

El elemento <ruleset> contiene uno o varios atributos <translate> y <dont-translate>.  Se pueden utilizar varios elementos <ruleset> para definir las reglas de varios tipos de documentos (esquemas) en el mismo archivo de reglas.

Elemento <translate>

El elemento <translate> especifica los elementos o atributos que se deben extraer para la traducción.

Atributo Descripción Obligatorio

 

Atributo

Descripción

Obligatorio

path

Expresión XPath para encontrar los elementos y atributos que se deben extraer para la traducción.

Sí

Loctype

Tipo de traducción. Especifica si los datos extraídos se pueden traducir o adaptar. Los valores posibles son "translatable" (traducible) y "localizable" (adaptable). El valor predeterminado es "translatable".  Todas las palabras de un elemento traducible se incluyen en el recuento de palabras del archivo.  Se asigna a los elementos adaptables un cómputo de una sola palabra, aunque contengan varias.  Un ejemplo de un elemento adaptable sería la ruta de acceso a una imagen original que, aunque haya que cambiarla, no debe incluirse en el recuento de palabras.

Optativo

datatype

El formato de los datos. Si los datos extraídos necesitan mayor extracción, debe especificarse su formato en este atributo. Constituye un caso típico el hecho de que un fragmento de HTML se almacene en un elemento XML. Si el atributo datatype tiene el valor "html", el extractor de XML extrae el contenido del elemento y abre el extractor de HTML que pasa el contenido extraído.  Observe que para que el código HTML que se halla dentro del XML se interprete correctamente, los ángulos < y > de las etiquetas HTML deben encerrarse en XML con sendos ángulos < y >, respectivamente.

Optativo

Type

El tipo de datos. Este atributo se utiliza cuando es preciso especificar explícitamente el tipo de datos que se van a extraer. Estos tipos se pueden excluir utilizando la función de exclusión de elementos en el perfil de traducción.  Puede incluir un tipo de elemento que ya existe o especificar un tipo de elemento nuevo y excluirlo cuando sea necesario.

Optativo

inline

Este atributo indica si los elementos especificados en el atributo "path" rompen un segmento. Si un elemento rompe un segmento, la etiqueta del elemento no se incluye en los datos extraídos. Si un elemento no rompe un segmento (si la etiqueta está alineada), la etiqueta del elemento se incluye en los datos extraídos. Los valores posibles para el atributo son "yes" o "no". Aquel significa que las etiquetas no deben romper los segmentos y este significa que las etiquetas sí deben romperlos. El valor predeterminado es "no".  Por ejemplo, si un elemento aparece en el medio de una oración, debería estar marcado como "yes", ya que en caso contrario la oración quedaría truncada en dos partes -separada por el elemento- lo cual sería incorrecto desde el punto de vista lingüístico.

Optativo

 

Elemento <dont-translate>

El elemento <dont-translate> especifica los elementos o atributos que no deben extraerse. Utilice este elemento en cualquier elemento extraído automáticamente que no deba traducirse.

 

Atributo

Descripción

Obligatorio

path

Una expresión XPath para asignar los elementos y atributos que no deben extraerse para la adaptación.

Sí

XPath

El atributo "path" de los elementos <translate> y <dont-translate> utiliza la expresión XPath. El lenguaje XPath permite direccionar las partes de un documento XML. Es una recomendación W3C. Este documento no trata las características del lenguaje, sólo se limita a dar un par de ejemplos. Puede leer la recomendación W3C en http://www.w3.org/TR/xpath.html. También puede encontrar un programa didáctico en la siguiente dirección web: http://www.arbortext.com/think%5Ftank/norm%5Fs%5Fcolumn/issue%5Fone/body%5Fissue%5Fone.html. Tenga en cuenta que esto se redactó antes de la publicación de la recomendación. Puede también examinar esta página web que contiene ejemplos de las expresiones XPath: http://www.zvon.org/HTMLonly/XPathTutorial/General/examples.html.

Regla predeterminada

Si un documento o elemento XML no está asociado con una regla, se aplica la regla predeterminada para extraer el documento.

  • En este caso, el contenido de todos los elementos se extrae como traducible.

  • Todas las etiquetas de los elementos rompen los segmentos. En otras palabras, todas las etiquetas pasan al esqueleto y no se incluye ninguna de ellas en un segmento extraído traducible.

  • No se extrae ningún atributo.

 

Codificación de caracteres

Al igual que sucede cualquier documento XML, a menos que el archivo esté codificado con UTF-8, se debe incluir en la declaración de XML la codificación de caracteres empleada en el archivo de reglas.

 

Ejemplo:

<?xml version="1.0"?>

<schemarules>

  <ruleset schema="ejemplo">

    <!-- No se extrae el elemento nombre bajo ejemplo -->

    <dont-translate path="/ejemplo/nombre"/>

    <!-- no se extraen los elementos con el atributo xlate y el valor "no" -->

    <dont-translate path=//*[@xlate=no]/>

    <!-- No se extrae nada bajo fragmento3 -->

    <dont-translate path="/ejemplo/fragmento3"/>

    <dont-translate path="/ejemplo/fragmento3//*"/>

    <!-- Se extraen todos los atributos bajo fragmento4 -->

    <translate path="/ejemplo/fragmento4/@*"/>

    <!-- Se extrae la etiqueta fragmento4 -->

    <translate path="/ejemplo/fragmento4" inline="yes"/>

  </ruleset>

</schemarules>

 

Definición de tipo de documento (DTD) empleada para archivos de reglas XML

<!-- schemarules.dtd creado el 31 de agosto del 2000 versión 1.0 -->

<!-- Raíz del documento -->

<!ELEMENT schemarules (ruleset)+>

<!-- Un elemento ruleset para cada tipo de documento. Puede haber múltiples elementos ruleset. -->

<!ELEMENT ruleset (translate|dont-translate)+>

<!ATTLIST ruleset

schema         CDATA #REQUIRED>

<!-- Especifique un elemento que se debe traducir. -->

<!ELEMENT translate EMPTY>

<!ATTLIST translate

path CDATA #REQUIRED

loctype (translatable|localizable) "translatable"

datatype CDATA #IMPLIED

type CDATA #IMPLIED

inline (yes|no) "no">

<!-- Especifique un elemento que no se debe traducir. -->

<!ELEMENT dont-translate EMPTY>

<!ATTLIST dont-translate

path CDATA #REQUIRED>
