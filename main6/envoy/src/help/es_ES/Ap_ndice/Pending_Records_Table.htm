Tabla de registros pendientes

Resumen:

Por cada tabla que contiene material traducible, es preciso crear una Tabla de registros pendientes en

la base de datos de origen. Un valor de importación de la base de datos de

System4 supervisa cada tabla de registros pendientes. Cuando se introduce un nuevo registro en la tabla de registros pendientes, el valor de importación

de la base de datos invoca un perfil de base de datos para determinar qué contenido se debe introducir en System4

para traducir ese registro. Luego, el registro se elimina de la tabla de registros pendientes.

 Cuando introduzca un nuevo registro en esta tabla, utilice los siguientes valores:

  • Task_Queue_ID NULL. Este campo incrementa automáticamente a medida que se introducen nuevos registros.

  • Record_Profile_ID El identificador del perfil de la base de datos de System4. El perfil de la base de datos contiene información acerca de los campos que deben traducirse en el registro y de los procesos de traducción que deben seguirse para el contenido.

  • Substitution_Parameters La clave primaria del registro. Si se trata de una clave compuesta, utilice cuatro plecas ( | ) para separar los elementos. Por ejemplo, en una clave que contiene un Item_ID y un Locale_ID, tendría que escribir así los valores de dichas claves: 1234 |||| en_US. Aunque no importa el orden en que aparecen las claves, si se utilizan métodos distintos para grabar las claves primarias en esta tabla, es importante que todos los métodos graben los elementos clave en el mismo orden.

  • Loaded 0. System4 emplea este campo para determinar si el contenido del registro ya ha sido importado en System4. El valor inicial de 0 indica que el registro no ha sido importado. Después de la importación, este valor se cambia automáticamente a 1 y si el valor es 1, se borra el registro. De esta forma System4 ocupa poco espacio en la base de datos.

 

Grabar registros en la tabla de registros pendientes (disparadores)

Se pueden utilizar varios métodos para grabar los registros traducibles en la tabla de registros pendientes.  Un método común consiste en utilizar disparadores. Los disparadores se incluyen en las tablas que contienen los datos traducibles. Siempre que se actualice o se introduzca un registro de origen, el disparador introduce un nuevo registro en la tabla de registros pendientes adecuada.

 

Otro método común consiste en utilizar el mismo mecanismo que su empresa emplea en la actualidad para colocar nuevos elementos en las tablas. Por ejemplo, si hay una aplicación personalizable que permite a los usuarios introducir nuevos productos en la base de datos, podría añadir a la aplicación una casilla de verificación rotulada "Traducir este registro". Si la casilla está marcada, cuando la aplicación graba el registro nuevo o cambiado en la base de datos, también lo colocaría en la tabla de registros pendientes adecuada para que pase a traducción. Este método permite a los usuarios especificar los cambios hechos en el registro que no necesitan traducirse. Por ejemplo, si un usuario actualiza un registro para corregir un error ortográfico, podría dejar la casilla sin marcar para que no vuelva a traducirse.
