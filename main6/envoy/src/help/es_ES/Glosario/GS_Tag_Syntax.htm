Sintaxis de las etiquetas GS

Las etiquetas GS se especifican en la sintaxis XML. Por lo tanto, todos los atributos tienen valores y todos los valores están entre comillas. No obstante, el extractor de HTML puede aceptar una sintaxis simplificada similar a HTML y normalizarla internamente a la sintaxis XML. La sintaxis simplificada, que se explica más adelante, no requiere que los valores de atributos estén entre comillas. Los valores de idioma (región) siguen el formato "es_ES".  Dentro de System4, los nombres aparecen en mayúsculas, por lo que deben escribirse en mayúsculas tanto los nombres como las referencias a ellos.  Los nombres también tienen un límite de longitud de 40 caracteres. Si epecifica un nombre más largo, quedará truncado cuando se añada a System4.

 

<GS ADD="nombre" />

Una posición donde se puede introducir un fragmento.

 

<GS DELETE="yes"> </GS>

Una región de contenido que se puede eliminar.

 

<GS DELETE="cualquier valor"> </GS>

Sintaxis simplificada; cualquier valor de DELETE se normaliza a "yes", incluso si es "no".

 

<GS DELETE> </GS>

Sintaxis simplificada; el valor de DELETE se establece en "yes".

 

<GS DELETE="yes" DELETED="idioma, idioma, "> </GS>

Una región de contenido que se puede eliminar y ha sido eliminado en algunos idiomas. Si el valor de DELETED está vacío, la región se incluye en todos los idiomas.

 

Estas etiquetas no se deben escribir nunca manualmente en las páginas, ya que dependen de la base de datos de System 4 y de la existencia de determinados fragmentos específicos. En su lugar, para introducir fragmentos o eliminar contenido utilice la interfaz de usuario del editor de fragmentos, que puede abrirse desde el editor principal.

 

<GS NAME="nombre" ADDED="idioma" />

Inserción de un fragmento genérico.

 

<GS NAME="nombre" ADDED="idioma" ID="núm" />

Si ID > 0, indica la inserción de un fragmento específico. Si ID = 0, se introduce el fragmento genérico.

 

Para cargar los fragmentos en la Biblioteca de fragmentos, se debe crear un archivo de importación de fragmentos en el que estos deben figurar entre las siguientes etiquetas.

<GS EXTRACT="nombre" [DESC="descripción"]> </GS>

Un fragmento genérico que se puede extraer.

 

<GS EXTRACT="nombre" locale="xx_XX" [DESC="descripción"]> </GS>

Un fragmento específico que se puede extraer.

 

Anidar etiquetas GS

Las etiquetas GS se pueden anidar.

  • Los fragmentos que se incluyen mediante una etiqueta GS ADDED puede contener posiciones de inserción y regiones eliminables.

  • A su vez, las regiones eliminables pueden contener posiciones de inserción y regiones eliminables. Si se elimina una región externa, se omiten las etiquetas GS incrustadas.

 

Corresponde a la página de origen y al autor del fragmento asegurar que las posiciones hagan referencia a nombres de fragmentos existentes, que todos los fragmentos, incluidos los de contenido HTML, se compongan de elementos de bloque de HTML, y que al introducir fragmentos o eliminar regiones se produzca código HTML válido.
