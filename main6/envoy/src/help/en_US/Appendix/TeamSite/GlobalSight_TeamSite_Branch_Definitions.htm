GlobalSight TeamSite Branch Definitions

You need to determine if your implementation of TeamSite requires the definition of GlobalSight TeamSite Branch Definitions by answering yes to one of the following questions:

·   Do you use the same TeamSite branch where you store your source content to store your localized files? If yes, then use the Default GlobalSight/TeamSite Export Operation. 

·   Do you use a new TeamSite branch to store your localized files? If yes, then use the Non-Default GlobalSight/TeamSite Export Operation. 

Before you can answer yes to one of the questions, however, you need to know how GlobalSight modifies the concept of TeamSite branches, and what are the default and non-default GlobalSight/TeamSite Export operations.

Default GlobalSight/TeamSite Export Operation

In your TeamSite application, you build branches, which are special TeamSite directories that contain working sub-directories and working files.  GlobalSight adds to the concept of TeamSite branches by assigning TeamSite branches with one of the following GlobalSight branch names:

·   TeamSite Combined Branch (Default Mode)

·   TeamSite Source Branch (Non-Default Mode)

·   TeamSite Target Branch (Non-Default Mode)

In the Default GlobalSight/TeamSite Export Operation, a TeamSite branch remains an unchanged directory structure as defined in TeamSite for regular work.  GlobalSight treats this unchanged TeamSite directory structure as a source for the content to be localized.

When you implement the Default GlobalSight/TeamSite Export Operation, GlobalSight understands the directory structure at the time of import into GlobalSight, and GlobalSight automatically creates a replicate of your Workarea directory structure under each TeamSite branch directory where you store content that is to be localized FOR THAT EXPORTED FILE.  GlobalSight names the replicate directory with the name of the target locale.  GlobalSight automatically inserts the localized content into the parallel locale directory that matches the Workarea source directory where the source content is stored.  This newly created combined directory structure is now referred to as a TeamSite Combined Branch, because the branch now contains both the source files for localization, and the target files that have been localized. 

The directory name of the TeamSite Combined Branch is unchanged in TeamSite.  For example, if the original branch name was Marketing, the name of the TeamSite Combined Branch would also be Marketing, but you would now find new sub-directories at the Workarea level below the branch directory, one new sub-directory for each target locale.

Example of a Complete TeamSite Directory Structure in the Default Mode

Let’s assume that you have two TeamSite branches before you start to work with GlobalSight, called A and B, each with a ‘Marketing’ Workarea. 

A (TeamSite Combined Branch)

  Marketing (TeamSite Workarea)

B (TeamSite Combined Branch)

  Marketing (TeamSite Workarea)

Since you have opted to implement GlobalSight with the Default GlobalSight/TeamSite Export Operation, you continue without making any changes to the branch directory structure (the changes will be made automatically by GlobalSight). 

Let’s further assume that you are going to translate the content in A to three target locales (Greek, French, Japanese), and translate the content in B in five target locales (Spanish, Yugoslavian, Greek, French, and Japanese). 

GlobalSight would automatically add sub-directories to your TeamSite Combined Branch at the time that you export the localized content from GlobalSight.  The resulting branch structure, after exporting some content in each target locale, would look like this:

A (TeamSite Combined Branch)

  Marketing (TeamSite Workarea)

  el-GR

  fr-FR

  ja-JP

B (TeamSite Combined Branch)

  Marketing (TeamSite Workarea)

  sp-SP

  su-YU

  el-GR

  fr-FR

  ja-JP

All of the behavior described above is at the branch/Workarea level.  However, there are some important considerations that occur in the Workarea sub-directory level, and we will now examine these considerations.

First, let’s assume that your Workarea (Marketing) has the following sub-directory structure:

A
  Marketing
    Directory1
      Directory11
      Directory12 (source files)
      Directory13

Now, let’s assume that, after performing a number of GlobalSight imports and exports, you add a sub-directory in the TeamSite Source Branch A.  Since GlobalSight automatically attaches a definition of the newly-changed source branch path to the incoming content to GlobalSight, when the content has been localized, the default GlobalSight/TeamSite Export Operation would reflect the change to the source directory structure, and automatically create the additional sub-directory in all affected locale sub-directories.

 

Finally, let’s assume that you delete a sub-directory in the TeamSite Source Branch B.  Since GlobalSight automatically attaches a definition of the newly-changed source branch path to the incoming content to GlobalSight, when the content has been localized, the default GlobalSight/TeamSite Export Operation would reflect the change to the source directory structure, and automatically create a new sub-directory in all affected locale sub-directories.

 

The Expected Behavior for the Default GlobalSight/TeamSite Export Operation

When using the Default GlobalSight/TeamSite Export Operation, GlobalSight imports TeamSite files from your TeamSite branch Workarea directory structure, and automatically creates a copy of the Workarea directory structure (directories only, not content of directories) thereby creating a TeamSite Combined Branch.  GlobalSight now exports the localized TeamSite files from GlobalSight into the TeamSite TeamSite Combined Branch directory structure in the parallel sub-directory to the branch directory under the Workarea directory where the source content is located.

 

 

Using the default GlobalSight TeamSite export behavior, you localize TeamSite content without further configuration of GlobalSight. 

Please note that temporary files are created in the same directory the files reside for Dynamic Preview. Those temporary files are deleted after the export of the files from GlobalSight to TeamSite.

Non-Default GlobalSight/TeamSite Export Operation

A TeamSite Target Branch is a special GlobalSight requirement that must be created ONLY if you are using the Non-Default GlobalSight/TeamSite Export Operation. 

The original TeamSite branch that contains the content to be localized is now referred to as a TeamSite Source Directory.  GlobalSight, in the non-default mode, does not change a TeamSite Source Directory in any way.  Whatever structure or whatever files are created for TeamSite in this branch remains unchanged.  All that is important for GlobalSight about this branch is that it contains the content to be localized.  When using the Non-Default GlobalSight/TeamSite Export Operation, you must identify this branch as the Source Directory to GlobalSight.

You must now create a new TeamSite Target Branch for every target locale you are anticipating will be used in GlobalSight.  You must duplicate the sub-directory structure of the TeamSite Source Branch exactly, including the exact character strings for all directory names.  You will probably need multiple TeamSite Target Branches for every TeamSite Source Branch, based upon the number of target locales, and you need to create a set of TeamSite Target Branches for every TeamSite Source Directory you have.

You have the freedom to name the new TeamSite Target Branch as you wish, but it is recommended that you name the new branch with the same name as the TeamSite Source Branch and append the target locale to it.  This will make the selection of the Source and Target Branches within GlobalSight very easy:

        TeamSite Source Branch Name:                               Marketing Materials

        TeamSite Target Branch Name:                               Marketing Materials in Greek

        TeamSite Target Branch Name:                               Marketing Materials in French

        TeamSite Target Branch Name:                               Marketing Materials in Japanese

Example of a Complete TeamSite Directory Structure in the Non-Default Mode

For example, let’s assume that you have two TeamSite branches before you start to work with GlobalSight, called A and B.  Since you have opted to implement GlobalSight with the Non-Default GlobalSight/TeamSite Export Operation, you need to create new TeamSite branches, one new branch for every target locale for the original TeamSite Branch.  Let’s further assume that you are going to translate the content in A to three target locales (Greek, French, Japanese), and translate the content in B in five target locales (Spanish, Yugoslavian, Greek, French, and Japanese).  The resulting branch structure would look like this:

A (TeamSite Source Branch)

A in Greek (TeamSite Target Branch)

A in French (TeamSite Target Branch)

A in Japanese (TeamSite Target Branch)

B (TeamSite Source Branch)

B in Spanish (TeamSite Target Branch)

B in Yugoslavian (TeamSite Target Branch)

B in Greek (TeamSite Target Branch)

B in French (TeamSite Target Branch)

B in Japanese (TeamSite Target Branch)

Now, let’s assume that you add a sub-directory in the TeamSite Source Branch A.  You must now immediately add this sub-directory using the exact character string for the directory name to the three TeamSite Target Branches associated with A, to ensure that the GlobalSight export operation does not encounter a different directory structure in the GlobalSight Target Directory from the GlobalSight Source Directory.

Finally, let’s assume that you delete a sub-directory in the TeamSite Source Branch B.  You must now immediately delete this sub-directory from each of the five TeamSite Target Branches directories for B.

The Expected Behavior for the Non-Default GlobalSight/TeamSite Export Operation

However, if your implementation of TeamSite does not use the TeamSite source branch directory for storing the localized content, then you must do two tasks:

·   Build a target branch for each target locale in your TeamSite environment that DUPLICATES the directory structure of the source branch in every way, including spelling the directory names exactly the same.  NOTE: This exact duplication is very important, because without it, GlobalSight cannot get the localized output of GlobalSight into the target directory.

·   Configure GlobalSight so that it understands your implementation of TeamSite source branch and target branch FOR EACH TARGET LOCALE.  You accomplish this by creating a definition for a source branch and target branch pair plus the target locale, and this definition is known in GlobalSight as a TeamSite Branches Definition:

THE GlobalSight TEAMSITE BRANCHES DEFINITION:

You add the first TeamSite Branches Definition to GlobalSight for the first target locale that applies to your TeamSite files.  You continue to add TeamSite Branches definitions to GlobalSight until you have one for every target locale you want to operate in the non-default GlobalSight mode.

After defining all the TeamSite Branches definitions, you begin to utilize GlobalSight from within your TeamSite application.  As you create TeamSite/GlobalSight Jobs, they carry the definition of the TeamSite Source Branch for each TeamSite file imported into GlobalSight. 

When the Jobs are completely localized and ready for export, GlobalSight remembers the Source Branch for each Page of the TeamSite/GlobalSight Job being exported.  GlobalSight then performs a match between the Source Branch of each Page and the Target Locale for that Page with all of the TeamSite Branches definitions.  If the exported Page values find a match within the one of the GlobalSight TeamSite Branches definitions, the matching Job’s Pages are exported into the Target Branch specified in the matching TeamSite Branches definition.

So, for example, the following sequences of events can occur during the matching between the Job’s Source Branch and Target Locale and any of the GlobalSight TeamSite Branches definitions:

·   Source Branch is a match, Target Locale is a match, and the Page is exported to the Target Branch specified (non-default GlobalSight behavior).

·   Source Branch is a match, Target Locale is not a match, and a new locale directory ‘copy’ is created in the TeamSite source directory and the Page is exported into that new directory (default GlobalSight behavior).

·   Source Branch is not a match, Target Locale is a Match, and a new locale directory ‘copy’ is created in the TeamSite source directory and the Page is exported into that new directory (default GlobalSight behavior).

·   Source Branch is not a match, Target Locale is not a Match, and a new locale directory ‘copy’ is created in the TeamSite source directory and the Page is exported into that new directory (default GlobalSight behavior).

Now, when GlobalSight’s GlobalSight understands which TeamSite Source Branch and Target Branch to use when there is a match with a GlobalSight TeamSite Branches Definition, it will know what to do:

 

 

If there is a change to the source branch directory structure, you must reflect this change in the target branch where your localized files are to be inserted. 

Please note that temporary files are created in the same directory the files reside for Dynamic Preview. Those temporary files are deleted after the export of the files from GlobalSight to TeamSite.
