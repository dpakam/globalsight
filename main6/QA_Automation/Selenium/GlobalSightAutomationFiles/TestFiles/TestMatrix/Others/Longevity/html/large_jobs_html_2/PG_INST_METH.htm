Instrument Method tab

Use this tab to include instrument method information. By default, all items are included in the Instrument Method report group, except Differences from Previous Version information.

Method/instrument information

Method Information – Displays the contents of the associated instrument method.

Instrument information to include:

  • Full Report – Generates a full report that includes all instrument information, channel information, and gradient and events tables.
  • Summary Report – Generates a report that includes channel and instrument names and other pertinent information in the Active Channels table for the associated instrument method.
  • Brief Report – Generates a report that includes the instrument names and other pertinent information in the Active Channels table for the associated instrument method.
  • None – Excludes any instrument information in the report.

Method revision information

Note: To report on this information, Full Audit Trail must be enabled for the project.

You can use the following method revision information:

  • Revision History Details – Displays a description of the change(s) made in the Revision Description field whenever you modified the instrument method (if Audit Trail Support is activated).
  • Revision Summary Information – Adds a table to the report that includes this information about the method: name, type, comments, date, modified user, locked status, method ID, and method version.
  • Differences from Previous Version – When reporting raw data, you can obtain a report of the method differences between the method version used to collect the data and the method version before the reported method version. When reporting using results, you can obtain a report of the method differences between the method version used to collect or process the data and the method version before it.

Select this check box to activate these option buttons and indicate the type of differences to include in the report:

  • Print Differences Only – Click this option button to include a table in the report that only includes information for fields that differ between the two methods.
  • Print Entire Methods – Includes a table that lists the entire contents of each method with differences highlighted, allowing you to see what fields have changed between the two versions.
