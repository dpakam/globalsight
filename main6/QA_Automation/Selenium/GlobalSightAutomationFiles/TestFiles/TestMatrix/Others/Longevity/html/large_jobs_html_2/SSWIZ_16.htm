New Sample Set Method template – Define User Prompts page

Use this page to define the prompts that you want to appear when a user creates a sample set method using this template. On this page, you also define the custom fields for which users can enter values, and you specify if the Component Editor is to appear when a user finishes creating a sample set method using this template.

Use this page to define the prompts that you want to appear when a user creates a sample set method using this template. On this page, you specify if the Component Editor is to appear when a user finishes creating a sample set method using this template.

Number of sample vials

Specifies if the Number of Sample Vials prompt is displayed when a user creates a sample set method using this template. When displayed, the field lets the user define the total number of sample vials in the tray for the sample set.

Vial starting position in tray

Specifies if the Vial Starting Position in Tray prompt is displayed when a user creates a sample set method using this template. When displayed, the field lets the user define the tray position number for the first standard or sample.

Select custom fields which user should enter

Specifies the custom fields that are to display when a user creates a sample set method using this template. When the custom fields are displayed, the user enters a value for each custom field. The custom fields listed in the box are those that are available for the current project.

Select fields which user should enter

Specifies the fields that are to display when a user creates a sample set method using this template. When these fields are displayed, the user enters a value for each field. The fields listed in the box are those that are available for the current project.

Display the component editor when a user finishes creating a sample set method using this template

Specifies if the Component Editor is to automatically appear when a user finishes creating a sample set method using this template. Note that when a user finishes creating a sample set method, the sample set method is automatically loaded into the Samples table and the Component Editor appears (if this check box is selected). The Component Editor lets you review or modify specific component information associated with the standards listed in the Samples table. Note that a user can access the Component Editor at any time from the Samples table (even if this check box is not selected).

Open Access options for 2690/95 or 2790/95

Choose the items you want to appear in the Open Access template wizard. The Open Access template wizard only appears when the Run Mode is set to Run Samples Using a Template. Choices:

  • Show Column Position column when loading – Select this check box to allow a Walk-up user to change the column position and view the sample function description before running samples.
  • Show Function column when loading – Select this check box to allow a Walk-up user to choose the appropriate sample function from the drop-down list and view the column position.
  • Show Method Set column when loading – Select this check box to display the Method Set/Reporting information. A Walk-up user can edit the Method Set.
  • Allow User to Insert/Remove rows when loading – Select this check box to allow a Walk-up user to edit the sample list by adding or removing samples. Use the Insert and Delete keys to add or remove rows respectively. The user can edit the Run Time column as well.

Open Access options for ZQ

This section applies when the ZQ Mass Detector option is installed. Choices:

  • Enter Masses – Select this check box to enable the Number of Masses field.
  • Number of Masses – Enter the number of masses on which to collect data. The valid range is from 1 to 20.

After making your selections, click Next to advance to the next wizard page.
