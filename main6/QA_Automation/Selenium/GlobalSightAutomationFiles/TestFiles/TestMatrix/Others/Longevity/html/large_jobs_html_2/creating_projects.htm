Creating projects

A project is a user-defined collection of methods, results, peaks, custom fields, and raw data.

Requirements:

  • Create Projects privilege
  • Create Project at the Root privilege (for a new top-level project)
  • To create validation template or validation working projects, you must have the Create Projects or Create Project at the Root privilege and either the Alter Any Project or Alter Project Type privilege.

To create a project:

  1. From the New Project Wizard - New Project Parent page, complete these tasks:
  • Determine whether the project will be in the project root directory or is part of a project hierarchy, as a sub-project to a parent project. This hierarchical structure allows you to organize multiple projects into one common project structure. When you back up a parent project and the sub-projects, the hierarchical structure remains intact.
  • Select the location in your project hierarchy where you want to create the new project. To create the project as a new top-level project, click the Projects folder. To create the project as a sub-project of an existing project, click an existing project folder. The existing project is designated as the parent project, and the new project is created as a sub-project of the parent.
  • Select the project type: Standard, Validation Working, or Validation Template.
  • If the Method Validation option is not installed, this field does not display and your projects are of the Standard type.
  • If the Method Validation option is installed, select either a standard, validation working, or a validation template project type. If you are collecting data for a validation study, select a validation working project. If you want to store the protocol methods and other methods required to validate your chromatographic methods but not collect data for validation studies, select a validation template project.
  1. Click Next.
