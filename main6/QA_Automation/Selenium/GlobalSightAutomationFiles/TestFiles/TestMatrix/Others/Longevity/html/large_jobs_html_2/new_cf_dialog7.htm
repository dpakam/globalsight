New Custom Field wizard - Translation Definition Table page

Use this page to view the entries in the Translation Definition table. This page appears only if the data type is Bool (Boolean) or Enum (Enumeration).

Tip: If you use the "Use As" feature in the New Custom Field Wizard – Source Selection page, you can use the translation table.

Boolean data type

If the data type is Bool, only two rows appear, one for each possible logical outcome of a Boolean expression.

Translation definition table for Boolean data type:

Column

Description

Value

Logical values for the Boolean custom field. For example, 0 (False), 1 (True).

Translation

Text translation you enter for each Value. For example, "Failed" for 0 (False) or "Passed" for 1 (True). This text appears in the custom field. Default: 0 = No, 1 = Yes in the Translation column.

 

Enum data type

If the data type is Enum and the field is calculated, the table contains one row for each comma-separated Boolean expression in the ENUM() function. In this case, you cannot add or remove rows. The Value column is read-only. The Translation column is initially empty. You enter the text translations for each Enum value.

If the data type is Enum and the field is not calculated (Data Source is External or Keyboard), you can add or remove rows as appropriate. You can also insert rows, drag and drop, and so on. Each time you add a row, the software recalculates the Value column to ensure the order of values increases starting from 0. Only the translations appear to move.

Translation definition table for Enum data type:

Column

Description

Value

Number of the enumeration field starting at 0.

Translation

By default, this column is blank. You can type appropriate translations as needed or edit existing translations. If you want blank to be a valid choice, leave this column blank. This text appears in the custom field.

Tips:

  • If you selected Use as Value on the Source Selection page, only numeric values can be used in the translation column.
  • If you selected Use as Field on the Source Selection page, you can translate the custom field to the value of another field by entering the name of the field followed by a space and the (fc) identifier. Custom fields can also be specified as the field name. Results for this type of boolean custom field are stored as strings, even if the field referred to using the field name (fc) syntax is a numeric field.

Example: Area (fc)
