Components tab

Use this tab to define how to calibrate standards (external or internal) and quantitate the components of interest in your unknown samples against known standards.

Global calibration parameters

The top section of the tab displays calibration parameters that relate to all components in the chromatogram.

Parameter

Description

Average by

How the system averages calibration standards to generate calibration curve data in the Calibration Curve window. After data processing, the calculated average response and/or average amount appear in the Calibration table of the Calibration Curve window. The averaged calibration curve points appear in the Calibration Curve plot of the Calibration Curve window.

How the system averages calibration standards to generate calibration curve data in the Calibration Curve window. After data processing, the calculated average response and/or average amount (or concentration) appear in the Calibration table of the Calibration Curve window. The averaged calibration curve points appear in the Calibration Curve plot of the Calibration Curve window.

Choices: None, Amount, Level

Update RT

Updates the peak retention times for the active channel to reflect the actual retention times of the peaks in the chromatogram. The updated retention times appear in the Calibration Status Results section of the Calibration Curve window.

Choices: Never, Replace, Replace Stds, Average, Average Stds

Update MT (CIA/CE option)

Updates the peak migration times for the active channel to reflect the actual migration times of the peaks in the electropherogram. The updated migration times appear in the Calibration Status Results section of the Calibration Curve window.

Choices: Never, Replace, Replace Stds, Average, Average Stds

RT window (%)

Retention time window for all components in the chromatogram as a percentage of each component’s retention time (RT).

Value range: 0.01 to 100%

Default: 5.00%

Default: 5.00% (1.00% for GC processing methods)

Tip: In the The RT Window field in the Components table, you can set the RT window on a component-by-component basis, overriding the setting in the global RT Window field.

When you click (Fill From Result), or add a new component and specify its retention time, the RT Window value of each component in the Components table is calculated as the product of the retention time of the component and the RT Window value. Any changes made to this parameter, therefore, are reflected in the Components table values.

MT window (%) (CIA/CE option)

Migration time window for all components in the electropherogram as a percentage of each component’s migration time (MT).

Entries: 0.01 to 100%

Default: 5.00%

Note: The MT Window field in the Components table also allows you to set the MT window on a component-by-component basis, overriding the setting in the global MT Window field.

When you click (Fill From Result), or add a new component and specify its migration time, the MT Window value of each component listed in the Components table is calculated as the product of the migration time of the component and the MT Window value. Any changes made to this parameter, therefore, are reflected in the Components table values.

CCalRef1

A known peak (in the chromatogram) for calculating interpeak custom results for all peaks in the Components table. This list contains the names of the components in the Components table. One custom calculation reference (CCalRef1) is available per processing method.

Note: In the custom component reference fields (CCompRef1, CCompRef2, and CCompRef3) in the Components table, you can define a reference peak (on a component-by-component basis) for calculating interpeak custom results. You can select a reference peak from the list. The CCompRef1, CCompRef2, and CCompRef3 fields are used in addition to the CCalRef1 parameter.

CCalRef1 is included in the Edit Custom Field Formula dialog box (in Configuration Manager) for calculating peak and interpeak values. When you use CCalRef1, set up your custom field formula before you define components in your processing method.

Include internal Std amounts in % amount calculation

Specifies whether to include the amounts for internal standard components identified in the Components table when calculating % Amount in the Peaks table (of the Review Main window and Results window).

Specifies whether to include the amounts for internal standard components identified in the Components table when calculating % Amount in the Peaks table (of the Review Main window and Results window) and the Peaks view table (of the Project window).

Sample value type

Specifies whether the values entered in the Components tab (either from the Component Editor or from the Default Amounts tab are concentrations or amounts. Empower software generates a calibration curve (Response as a function of Amount or Concentration) for all named components in the Components table. Choices:

  • Amount – The component values are entered as amounts (total quantity of the component in the chromatogram).

Note: If developing a processing method for dissolution data, set the Sample Value Type parameter to Amount.

  • Concentration – The component values are entered as concentrations (amount of component in a specified volume).

Tip: By using the Sample Value Type and X Value fields in the Components tab, Empower software can quantitate by injection volume.

Component table

Use the Components table to define component-specific parameters. Each row in the table represents a component to identify. After a standard is integrated, click (Fill Table from Result) to fill the Components table using information from the selected chromatogram in the 2D Channels table. By default, peaks are initially named Peak1, Peak2, and so on. From that point on, you can fill in the Components table based on this initial component information.
