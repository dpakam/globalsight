New Project Wizard - Tablespace page

Use this page to specify these settings for this project:

  • the amount of tablespace you want to allocate
  • full audit trail
  • data processing integration, ApexTrack or Traditional

TableSpace

Define the amount of tablespace reserved in the database for the project. After you create the project, you can increase or decrease the tablespace quota in the Project Properties dialog box if you have the Alter Any Project privilege.

Tips:

  • The minimum project quota is 50 MB.
  • A project's tablespace is allocated from the main project tablespace, which is controlled by settings in the Database Properties dialog box.

TableSpace available

The amount of unused tablespace that is available to the project.

Full Audit Trail Support

Empower automatically tracks system-level activities using secure, time-and-date stamped audit trail records. If desired, you can also set up Empower to monitor the history of certain project-level activities, including the creation, modification, or deletion of objects, such as samples, methods, and results. When you create a project, you determine the full audit trail setting. The full audit trail setting is preserved when you back up, clone, or restore a project.

Restrictions

The following New Project System Policies (View > System Policies > New Project Policies) affect whether you an change the default Full Audit Trail settings when creating a project.

  • If the "Don't Allow User to Change Default Full Audit Trail Support Setting" is enabled, you cannot change the default setting for "Full Audit Trail Support" when creating a project.
  • If the "Don't Allow User to Change Default 'Require User Comments On' Setting" is enabled, you cannot change the default settings for comment and password requirements.  

If you have the View Audit Trail privilege, you can view the project audit trail in the Audit Trails tab of the Project window.

To specify that users enter comments and/or their password when they create, modify, and/or delete a method, result, or sample within a project:

  1. In the Project Audit Trail Policies table, click the Comment column for the Method, Result, Sample, or Deletion row, and then select one of the following choices:
  2. To require that users do not enter a comment when they create, modify, and/or delete a method, result, or sample, select Silent.
  3. To require that users enter a pre-defined comment when they create, modify, and/or delete a method, result, or sample, select Restricted.
  4. To require that users enter a free form comment when they create, modify, and/or delete a method, result, or sample, select Unrestricted.
  1. To require that users enter their password when they create, modify, and/or delete a method, result, or sample, select the Confirm Identity checkbox.

Data Processing Techniques

Choose the type of data processing integration, ApexTrack or Traditional, to be used as the default algorithm.

Requirement: You must enable ApexTrack before you can use it in your project. Select View > System Policies and then select the Data Processing tab. Select the option to Allow the use of ApexTrack Integration.
