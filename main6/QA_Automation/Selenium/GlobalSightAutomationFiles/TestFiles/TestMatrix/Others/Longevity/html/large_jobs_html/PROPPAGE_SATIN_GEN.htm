General tab (busSAT/IN and busSAT/IN 2)

Use this tab to define channel description and analog signal conversion parameters for detection devices connected to busSAT/IN Module channels.

Use this tab to define channel description, analog signal conversion parameters, and BCD parameters for detection devices connected to busSAT/IN Module channels.

Channel name

Software-designated name for the channel, appears in the Active Channels table in the Instrument Method Editor (displayed in Run Samples).

Description

Optional description for the busSAT/IN channel, up to 250 alphanumeric characters. The description appears in the Active Channels table in the Instrument Method Editor and in the Real-Time Plot pane in Run Samples.

Channel enable (busSAT/IN 2 only)

Specifies the data channel as an active channel for the busSAT/IN 2. Clear this check box if the channel is unused.

Choices: Enabled or Disabled

Default: Enabled

Sampling rate

Rate used to collect data for this channel. Set a sampling rate to achieve at least 15 data points across the narrowest peak of interest.

Rate used to collect data for this channel. Typically, for LC, IC, and GC, set a sampling rate to achieve at least 15 data points across the narrowest peak of interest.

Choices: 0.25, 1, 2, 5, 10, 20, 25, 50, or 100 points per second

Default: 1

Note: The maximum sampling rate for all detectors connected to the busLAC/E interface card is 400 points per second. Higher sampling rates produce more data points and require a greater amount of disk space.

Units

Units that appear on the y-axis of the chromatogram (for example, mV or AU). Units multiplied by Scaling Factor equal 1 V. The maximum Units label length is six alphanumeric characters. The default is mV.

Scaling factor

Value that the software uses to adjust the display of the signal along the y-axis of the chromatogram. This value is used to associate the Units parameter to volts (V). Scaling Factor multiplied by Units equals 1 V. For example, if you require AU on the y-axis and are using a detector where 1 AU equals 1 V, enter a Scaling Factor of 1.0.

Default: 1000.0

BCD input enabled

Specifies if Binary Coded Decimal (BCD) inputs from a non-IEEE-488 autosampler are included as part of sample information. BCD information records the vial number from which an injection is made.

Choices: Enabled or Disabled

Default: Disabled

Vial confirmation and error message

When the autosampler completes an injection, it sends vial number information to the busSAT/IN Module and into the Barcode/BCD field in the database. If the vial number sent from the autosampler:

  • Matches the vial number entered in the Samples table in Run Samples, no additional information is stored in the database
  • Does not match the vial number entered in the Samples table in Run Samples, a message (indicating the vial number from which the injection was made) is stored in the Error Log of the database for this injection

Note: To include BCD information in reports, include the Barcode/BCD field in the associated report method. You can find the Barcode/BCD field in the Injection branch of the Fields report group in Report Publisher.

BCD polarity

Polarity of the BCD input signal:

  • Positive polarity (+) - Choose this setting when an open circuit represents 1, and 0 V represents 0.
  • Negative polarity (–) - Choose this setting when an open circuit represents 0, and 0 V represents 1. This is the default setting.
