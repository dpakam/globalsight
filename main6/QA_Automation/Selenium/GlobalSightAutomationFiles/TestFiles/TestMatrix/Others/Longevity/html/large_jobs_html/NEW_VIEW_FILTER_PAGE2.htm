New View Filter wizard - Conditions page

Use this page to set conditions (optional) in the view filter columns to further filter data that is displayed in a view. A condition typically consists of an operator (for example, >) and a reference value (for example, 1/23/2001) that, in combination, limit the data that appears in the view to which the view filter is applied.

Conditions table

Displays columns for Column Names (previously selected in the first page) and up to five conditions that you can set for each Column Name.

To set view filter conditions (optional) for a Column Name:

  1. Click the Condition 1 cell in the row of a Column Name to which you want to add a condition. The Create a Condition dialog box appears.
  2. If applicable, select an operator (for example, Greater Than Equal) in the first field.

Note: Some Column Names do not have an operator field (for example, Sample Type).

  1. In the next field, enter or select a reference value (for example, 3/1/2001).

Note: If you select Between as an operator, the dialog box activates the second reference field. Enter values in the two reference fields to set a range for the search.

  1. Click OK. The dialog box closes and the condition is added to the cell.
  2. If you want to broaden (logical OR) the search conditions for the column (for example, to search for Broad Standards and Broad Unknowns in a Sample Type column), repeat steps 1 to 4, but click the next available Condition cell.

Rule: You can enter up to five conditions for a Column Name.

  1. Repeat steps 1 to 5 for each Column Name to which you want to add a condition.
  2. Click Next.
