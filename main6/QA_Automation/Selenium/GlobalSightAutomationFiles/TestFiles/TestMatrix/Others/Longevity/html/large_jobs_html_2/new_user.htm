New User dialog box

Users accounts are controlled by either Local or LDAP authentication. Both forms of authentication require users to enter their Empower user name and password when they log in to Empower, signoff on a result, or re-enable a locked session. Local Empower user accounts are governed by the User Account Policies (Accounts and Passwords) tab in the System Policies dialog box. LDAP user accounts are governed by the policies on the LDAP server. Both LDAP and Local user accounts contain unique information for each user. For user accounts governed by LDAP authentication, users' LDAP log in name and password are usually identical to their Empower user name and password.

Rule: The System account is always a Local account and cannot be changed.

When you create a user, Empower stores required information for that user within a unique user account. All newly created user accounts are active. The new user's privileges depend on the user type you assign to that user.

The System Administrator defines and maintains most user settings (account status, user type, access to Empower's interfaces); however, users with Local accounts can change their own password. Users with Administrator or Alter User privileges can change other users' passwords. For LDAP user accounts, users' passwords are maintained on the LDAP server.

Requirement: Create Users or Administrator privilege

To create a new user account in Local authentication mode:

  1. In the New User dialog box, enter the information desired for the user's account, keeping the following guidelines in mind:

Option

Guideline

User Name

The user's name must be a minimum of 4 characters and a maximum of 30 alphanumeric or underscore characters.

Exception: LDAP user accounts can be a minimum of 3 characters and a maximum of 30 characters.

Restriction: When the Unique User Account Names system policy is enabled, the name you specify must be unique within the Empower database.

Full Name

Optional. When you want to include the user's full name on reports, specify the user's first and last name.

Exception: A full name is required for users who must signoff on results in Report Preview.

Allowed User Types

Control the user's access rights and software privileges by selecting user types that the user can choose from.

Requirement: You must select at least one user type for a new user, however, you can select multiple user types that the user can choose from.

Example: If John Smith is assigned multiple user types, such as analyst and chemist, when he logs in as a chemist, he only has chemist privileges. As a chemist, he can lock and unlock projects, but he cannot perform these tasks when he is logged in as an Analyst unless the analyst user type was modified. When a user possesses multiple user types, you can determine which user type was employed to perform a task by viewing the system audit trail and/or project audit trail records.

Default User Type

A default user type for this user based on their tasks and responsibilities.

Requirement:  You must select a default user type from the list.

Password Required

When required, the user's password is the same as their user name until the user or the system administrator changes the password. Empower prompts the user to change his or her password the first time he or she logs in.

Restriction: If the Unique User Passwords system policy is enabled, users must have a unique password and cannot reuse an old password. If the Passwords Expire every XXX days system policy is enabled, the user's passwords will expire every x number of days as specified in this policy. If the Minimum Password Length system policy is enabled, the user's password must be at least the number of characters specified in this policy.

Always Local Login

Before enabling LDAP authentication, you can specify that this user account remain a local account and not be associated with an LDAP account.  When you select this check box and enable LDAP Authentication, this user account is still governed by the User Account Policies (Accounts and Passwords) tab in System Policies and not the policies on the LDAP Server.  If you do not select this check box and LDAP Authentication is enabled, the user account must be associated with an LDAP account so that the user can log in to Empower.

Tip: This field is available in Local authentication mode only.  

Default User Interface

The chosen Empower interface automatically appears when the user logs in to Empower. The user can select a different interface by clicking the Advanced button on the Logon dialog box.

Tip: If you select the system default interface, the interface specified in the User Account page of the System Policies dialog box appears when the user logs on.

Allowed Interfaces

If you want to give this user access to more than one Empower interface, select the additional ones. If you do not select an interface, the user will log onto the default user interface.
  1. Click OK.
  2. Supply a comment and/or a password, if required, and then click OK.

Result: The new user's name appears in the Users table.

To create new user account(s) in LDAP authentication mode:

  1. In the Select users from the LDAP tree dialog box, select multiple user names or an individual user name from the LDAP Directory tree, click Add Names , and then click Create Users.
  2. In the New User dialog box, enter the information desired for the user's account, keeping the following guidelines in mind:

Option

Guideline

User Name

This field will be blank if multiple users are created simultaneously. The Empower user name defaults to the user's LDAP login name.

Full Name

This field will be blank if multiple users are created simultaneously. The Empower user's full name defaults to the user's LDAP first and last names.

Requirement: A full name is required for users who must signoff on results in Report Preview.
