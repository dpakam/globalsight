General Tab (User)

You can modify your own password (unless this is an LDAP user account) and set your own table and plot preferences without the Alter Users privilege.

Restriction: System is a permanent user that cannot be deleted from the database. The System account is always a Local Authentication user account.

User Name

Name of the selected user.

Full Name

Complete name of the selected user.

Tip: This field is optional. However, if the user needs to sign off results in Preview, you must make an entry in this field. The field can also be used to print a user's name on a report. You need the Alter Users privilege to modify this field.

Account Status

Current condition of the selected user account. Choices:

  • Active – Specifies that the selected user account is operating normally.
  • Disabled – Specifies that the selected user account was deactivated by the software and cannot be used to log in to Empower software. This status appears in the Users View table when, for example, a user exceeds the allowable number of login attempts.

Tip: If the number of user accounts with a status of Active or Disabled exceeds the number of licenses installed, then change the status of a user to Removed before creating a new user. If you cannot remove a user, you must buy additional licenses.

  • Removed – Specifies that the selected user account was deleted from the current database and the account cannot be used to log in to Empower software.

Tip: When you delete a user and Enforce Unique User Account Names is selected in the System Policies dialog box, Removed appears in the User Status column of the Users View table. A user with Administrator privileges can later activate a user account with a status of Removed. This is useful if, for example, a user terminates employment at your company and is rehired three months later.

LDAP user accounts

If you have LDAP Authentication enabled (View > System Policies > Authentication Policies) the password fields are disabled because it is controlled by LDAP and not by Empower software.

Exception: When the Always Local Login check box is selected, the password fields are enabled because this user account is a Local account and not an LDAP account.

LDAP User DN

The LDAP user distinguished name is supplied by the LDAP server. If you want to associate a different LDAP user name with this Empower user account, click Search LDAP. You cannot modify this field.

Tip: This field displays when the user account is controlled by LDAP authentication and the Always Local Login check box was not enabled.

Password Required

Specifies that the selected user must enter a password when logging in to Empower software. This password affects the ability for a user to sign off results in Preview.

Always Local Login

Specifies that the user account is always a Local account even though LDAP authentication is enabled. When this check box is selected and LDAP Authentication is enabled, this user account is still governed by the User Account Policies (Accounts and Passwords) tab in the System Policies dialog box and not the policies on the LDAP Server. If this check box is not selected and LDAP Authentication is enabled, the user account must be associated with an LDAP account so that the user can log in to Empower software.

Tip: This field does not display for any user accounts when in LDAP authentication mode.

Current Password

Specifies your current password. To change your password, ensure the Password Required check box is selected. Enter a new password in New and Confirm Password field.

New Password

Specifies a new password. The minimum length is specified by your system policies (up to 30 characters).

Confirm New Password

Requires that you reenter your new password to confirm your choice of password.

User Must Change Password at Next Login

Requires that the selected user must change his or her password the next time he or she logs on to Empower software.

Don't Show License Expiration Warning (User Preference Setting

Select this check box to turn off the warning message that displays seven days prior to the expiration date for time-based AQT licenses. If you want the license expiration message to display, clear this check box.
