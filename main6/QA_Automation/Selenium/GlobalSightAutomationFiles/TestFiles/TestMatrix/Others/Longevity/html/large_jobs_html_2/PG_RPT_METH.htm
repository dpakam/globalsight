Report Method tab

Use this tab to identify items to include in the report and information to include in the Revision Description field of the report. By default, all items are included in the Report Method report group, except the Differences from Previous Version information.

Note: You will only obtain this information when reporting on the method itself. This information is not reported for data, results, etc.

Items to include in report

You can include the following report items:

  • Method Information – Includes the report method information in the report.
  • Group Names – Includes the report groups in the report.
  • Extended Group Information – Adds to the report very detailed information about report groups, including settings for each item in all report groups.

Method revision information

Note: To report on this information, Full Audit Trail must be enabled for the project.

You can include the following method revision information:

  • Revision History Details – Adds a description to the report of the changes made in the Revision Description field whenever you modified the report method.
  • Revision Summary Information – Adds a table to the report that includes information about the method: name, type, comments, date, modified user, locked status, method ID, and method version.
  • Differences from Previous Version – Activates these option buttons to indicate the type of differences to include in the report:
  • Print Differences Only – Includes a table in the report that only includes information for fields that are different between the two methods.
  • Print Entire Methods – Includes a table that lists the entire contents of each method with differences highlighted, allowing you to see what fields changed between the two versions.
