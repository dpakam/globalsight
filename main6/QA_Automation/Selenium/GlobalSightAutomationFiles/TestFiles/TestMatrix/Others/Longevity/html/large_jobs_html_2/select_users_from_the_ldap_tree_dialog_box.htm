Select users from the LDAP tree dialog box

In this dialog box you can create one or more Empower user accounts by selecting LDAP user names. You can select multiple user name nodes from the displayed forests. When you click Create Users, one account is created for each user selected in the tree. All accounts are active Empower user accounts with the same user type (associated privileges) and Empower software interface access. The Empower user account's full name is identical to the LDAP full name and the user name is identical to the LDAP login name.

Prerequisite: LDAP Authentication enabled (View > System Policies > User Account tab > Authentication > LDAP)

You can access this dialog box by creating a new Empower user account (File > New User or right-click New > User).

Requirement: Create Users privilege or Administrator privilege

To create an Empower user account:

  1. Select multiple user names or an individual user name from the LDAP Directory tree and then click Add to move them to the New Users pane.

Tips:

  1. You can search for an LDAP user name by entering the user's name in the LDAP directory list field and then clicking Search . To remove a user from the New Users pane, click Remove .
  2. The LDAP Directory tree contains both active and disabled user accounts that reside on the LDAP server.
  1. Click Create Users.

Result: The New User dialog box appears but does not populate the User Name or Full Name data fields. Each account is created with the same user type(s) and Empower software interface access.

Exception:

The User Name and Full Name fields are blank unless one of these conditions apply:

  • an existing Empower user name matches the LDAP user name
  • the LDAP log in name contains invalid characters for an Empower user name
  • the LDAP log in contains too few or too many characters.

In any of these cases, you can edit the User Name of the problematic user account.
