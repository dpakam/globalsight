Slicing tab (GPC)

Use this tab to specify GPC/V/LS processing parameters that identify the broad peaks to be sliced for the molecular weight distributions.

Use this tab to specify GPC processing parameters that identify the broad peaks to be sliced for the molecular weight distributions.

Note: If you enable the Chemical Structures option, additional fields appear in this table and in the Peaks table.

Name

In the Component Editor, the identifier assigned to the broad peak when a distribution is calculated for the peak. This name originates from the Slicing table and is assigned to the peak if it is within the RT Window of the broad component (when no component is listed for the sample).

When a component is listed for the sample in the Component Editor during sample loading or in the Alter Sample window, the RT Window is ignored, and the names (instead of the times) are matched. A distribution is calculated for the peak whose sample component name and broad component name match.

Retention time (min)

Retention time at the center of the retention time window of the peak.

Retention time window (min)

Retention time window used in peak matching (where retention time window equals Retention Time ±RT Window).

Peak match

Peak matching type used to determine the peak within the retention time window that most closely matches the Broad Components table entry. The choices are:

  • Closest (default)
  • Greatest Area
  • Greatest Height
  • Greatest Width

dn/dc (ml/g) (GPCV or GPC/V-LS option)

Note: To perform LS processing, enter a dn/dc value for your standard or unknown in the Component Editor.

The known dn/dc for a broad peak (where dn/dc is the change in refractive index over the change in concentration associated with the refractive index detector in use). dn/dc allows you to use the dn/dc and RI detector response to calculate the concentration of the peak, assuming you have calibrated your RI detector using a narrow standard peak with known dn/dc and exact concentration. When you have two or more peaks in the same sample with different dn/dc values and you know that the peaks have different refractive indexes, you might have to enter dn/dc in the Slicing table. This instructs the software to calculate an appropriate concentration for the peak.

Entries: –10.0000 to 10.0000 mL/mg (with a precision of 4)

Default: Blank

Notes: You can enter a dn/dc value in the Slicing table for broad unknown and standard peaks, not for narrow standard peaks. For narrow standards, enter the dn/dc in the Weight Loading table of either the Component Editor or Alter Sample window. For unknowns (broad or narrow), you can also enter a single dn/dc value during sample loading or in the Alter Sample window.

For the calculation of dn/dc and concentration for GPCV data, fill in the RI Sensitivity field in the Samples table of Run Samples. Enter 1.0 if the "gain" for the RI detector never changes or enter the current setting from the RI detector to ensure that the numbers calculated are accurate.

The RI detector response for the narrow standard peak, along with its dn/dc and exact concentration, is used to calculate the RI Calibration Constant in the Peaks table of the Review Main window and Results window. The RI Calibration Constant (which is the change in refractive index divided by the change in detector response) is used to calibrate the RI detector response to obtain an absolute number. Otherwise, the concentration can be calculated for samples where only the dn/dc is known. With an absolute RI detector response, dn/dc can be calculated for samples where the concentration is known.

If you do not know the dn/dc value and have already calibrated your RI detector and calculated an RI Calibration Constant, enter a concentration during sample loading. Empower software calculates the dn/dc value and places it in a peak field called "dn/dc" in the Peaks table of the Review Main window and Results window. The associated "Calculated dn/dc" peak field is checked.

Notes:

  • Verify that the concentration detector is an RI detector in the GPCV channel set used to process the data. Set up a GPCV instrument method and verify that the concentration detector is in the instrument method.
  • If you do not enter a concentration, Empower software uses the RI Calibration Constant associated with the instrument method for GPCV and in LS calibration for LS.

A2 (10e-4 * mL * mol/g^2) (GPC/V-LS option)

The second virial coefficient. The default value, 0, is superceded by the value you enter during sample loading in the Component Table or Alter Sample. The range of valid A2 values extends from –100.000 to +100.000 with a precision of 3.

Rg Fit (GPC/V-LS option)

Fit type used to calculate the fitted radius of gyration, Slice Fitted Rg. The observed radius of gyration (Slice Observed Rg) values derived from the LS data are fit as a function of elution volume, Rg = f(v) either using a 1st through 5th Order polynomial function or a 1st to 2nd Order Step function as specified in this field to produce the Slice Fitted Rg values. The Rg data is always fit first.

Fitting model

How the intrinsic viscosity and molecular weight values are assigned across the peak. The Fitting Model controls whether the MW and Intrinsic Viscosity are fit against molecular weight or volume. Select one of the following:

  • Structural – For peaks whose viscosity profiles can be modeled using a viscosity law model where the observed intrinsic viscosity is fit as a function of molecular weight, y = f(log M). Select this model with there is a consistent correlation of log Rg to log M across the peak as when the peak is made up of a single polymer. Structural is the default.
  • Calibration – For peaks that are made up of a blend of polymers. When you choose Calibration, the observed intrinsic viscosity is fit as a function of elution volume, y = f(v). This model does not make any assumptions about how well log Rg correlates to log M across the peak. Calibration is the default when the LS option is enabled.

For GPC/V-LS, the Fitting Model and MW Fit fields work together to specify how the fitted molecular weight values are calculated.

For GPCV-LS, when the Fitting Model is set to Calibration, the Intrinsic Viscosity Fit, Rg Fit, and MW Fit can all be set to different fit types. When the Fitting Model is set to Structural, the Rg Fit and MW Fit can be set to different fit types only when the Intrinsic Viscosity Fit is not 1st Order.

MW Fit (GPC/V-LS option)

Fit type used to calculate the fitted MW, Slice MW. When the Fitting Model is set to Structural, the fitted Rg (Slice Fitted Rg) values and observed molecular weight (Slice Observed MW) values derived from the LS data are modeled using log Rg = f(log M) using either a 1st through 5th Order polynomial function, or a Zimm-Stockmayer fit as specified in this field. When the Fitting Model is set to Calibration, the observed molecular weight (Slice Observed MW) values derived from the LS data are fit as a function of elution volume, log M = f(v) using either a 1st through 5th Order polynomial function, or a 1st or 2nd Order Step function as specified in this field.

Intrinsic viscosity fit (GPCV or GPC/V-LS option)

Fit type of the curve fitted to the observed viscosity data. The data from the fitted curve is used to calculate the distribution of molecular weights, which, in turn, is used to calculate molecular weight averages. Different choices appear depending on whether you select Structural or Calibration as the Fitting Model. Choices:

  • 1st through 5th Order, Random, 1st Order step, and 2nd Order step curve fit.
  • Random Fit and Branching Model set to 3 Branch Points.
  • Random Fit and Branching Model set to 4 Branch Points.

You specify the Random intrinsic viscosity fit to apply based on the method used to synthesize the branched polymer. You can evaluate the results after applying each of the Random intrinsic viscosity fits.

When the LS option is enabled and the GPCV option is not enabled, the Intrinsic Viscosity Fit field (which does not appear in the table) is automatically set to the same fit as the MW Fit.

k LCB (dl/g) (GPCV or GPC/V-LS option)

Mark-Houwink constant. The intercept (log K) of the Viscosity Law plot. User-entered values that Empower software uses in long-chain branching calculation instead of calculating it.

Range: 0.000000 to 2.000000

alpha LCB (GPCV or GPC/V-LS option)

Mark-Houwink constant. The slope of the Viscosity Law plot. User-entered values that Empower software uses in long-chain branching calculation instead of calculating it.

Range: –1.000000 to 2.000000

k LCB (LS) (nm) (GPCV or GPC/V-LS option)

Mark-Houwink constant. The intercept (log K) of the Conformation plot. User-entered values that Empower software uses in long-chain branching calculation using LS data instead of calculating it.

Range: 0.000000 to 2.000000

alpha LCB (LS) (GPC/V-LS option)

Mark-Houwink constant. The slope of the Conformation plot. User-entered values that Empower software uses in long-chain branching calculation using LS data instead of calculating it.

Range: –1.000000 to 2.000000

Linear % of Data Region (GPCV or GPC/V-LS option)

The percentage of the data region (at the low-molecular weight end) to which a tangent line is fit to produce the K and alpha values for the distribution.

Default: 40%. If the Intrinsic Viscosity Fit is set to 1st Order and the Fitting Model is set to Structural, the Linear % of Data Region is set to 100% and cannot be changed. K and alpha are calculated by the linear fit, where log K is the intercept of the linear fit, and alpha is the slope.

If the intrinsic viscosity fit is one of the Random fits, the percentage is blank and cannot be changed. K and a (alpha) are calculated as part of the Random fits without using a percentage.

For GPC-LS, when the Fitting Model is set to Structural, the Linear % of Data Region field is editable only when the MW Fit is set to the higher-order polynomial fits (2nd through 5th Order and 1st and 2nd Order Step) and the alpha LCB (LS) field is blank. For GPCV, this field is not editable when alpha(LCB) is blank.

epsilon (GPCV or GPCV-LS option)

The value of the shape factor constant, epsilon, used in the calculation of the Random Intrinsic Viscosity Fit. This column is accessible only when a Random intrinsic viscosity fit value is used in the Intrinsic Viscosity Fit column.

Range: 0.500 to 1.500

Default: Blank

For GPCV-LS, this field is enabled when the processing method is universal.

Branching Model (GPCV or GPC/V-LS option)

How Empower software calculates d (the branching density, or the number of branches per Dalton). For GPCV:

  • If you chose a 1st through 5th Order intrinsic viscosity curve fit, the Branching Model choices are None (d is not calculated) or Star (d is calculated using the formula for a star-branched polymer).

Note: If the Branching Model is set to Star, d is calculated using this formula:

d = f/M
Where f = Number of arms in the star-branched polymer
M = Slice molecular weight

Default: None.

  • If you chose the Random fit type and Branching Model of 3 Branch Points or 4 Branch Points, d is calculated using this formula:

d = (f – 1)*lambda + (1/M)
Where lambda is a result of the Random fit
M = Slice molecular weight
f = Number of branch points (3 or 4)

Fitting Model choices:

  • When the Fitting Model is set to Calibration, the Branching Model field lists all choices; None (d is not calculated), Star (calculated as above), 3 Branch Points, or 4 Branch Points.
  • When the Fitting Model is set to Structural, the Branching Model field choices depend on the MW Fit choice.
  • When you set the MW Fit to 1st through 5th order polynomial function or 1st or 2nd step function, the Branching Model choices are None or Star.
  • When you set the MW Fit to Zimm-Stockmayer, the Branching Model choices are 3 Branch Points and 4 Branch Points.

Note: When the Branching Model is set to 3 Branch Points and 4 Branch Points, d is calculated using this formula:

d = (f – 1)*lambda + (1/M)
Where lambda is a result of the Zimm-Stockmayer MW Fit
M = Slice molecular weight
f = Number of branch points (3 or 4)

Type

Whether the polymer is found in a standard, unknown, or standard and unknown. Depending on the type, the software matches:

  • Standard components only when performing calibration
  • Unknown components only when performing quantitation
  • Standard and unknown components when performing calibration or quantitation

High MW (Daltons)

The highest molecular weight slice to be included in the calculation of the molecular weight averages for a broad unknown peak. Slices with molecular weights greater than this value are deleted from the molecular weight distribution before the molecular weight averages are calculated.

Low MW (Daltons)

The lowest molecular weight slice to be included in the calculation of the molecular weight averages for a broad unknown peak. Slices with molecular weights less than this value are deleted from the molecular weight distribution before the molecular weight averages are calculated.

Accumulate area

How the cumulative % area of the polymer is accumulated when the molecular weight distribution is calculated. The choices are:

  • High to Low (Default) – Starts accumulating area at the high-molecular weight side of the distribution.
  • Low to High – Starts accumulating area at the low-molecular weight side of the distribution.

Mol Wt Marker 1 through Mol Wt Marker 6 (Daltons)

Six molecular weights the software uses when calculating the percent area of the distribution that has molecular weights above or below the molecular weight marker. The molecular weights and percent area values appear in the Peak table of the Review Main window and the Results window.

Note: The first three percent area values (%Poly > MWM#) are calculated as the percent area of the distribution with molecular weights greater than the molecular weight associated with the molecular weight marker. The last three percentage area values (%Poly < MWM#) are calculated as the percent area of the distribution with molecular weights less than the molecular weight associated with the molecular weight marker.

Moment area, moment molecular weight

Calculates the molecular weight averages (moments) for a distribution custom field value in place of the standard Empower area or molecular weight values.

For example, you created the custom distribution field "AdjArea" to correct the area of a slice for its dn/dc dependence on molecular weight. Use this custom distribution field in place of area to calculate the molecular weight averages. This correction changes the calculated molecular weight averages, basing them on the adjusted shape of the peak.

Notes:

  • You can also plot the Adj Area on the Distribution plot with, or in place of, the normalized distribution dwt/d(logM).
  • Choices for Moment Area that the software uses for calculation:
  • Slice Area – The area of the peak slice (default).
  • Slice Conc – The concentration of the peak slice (GPCV and GPC/V-LS processing only).
  • Distribution Custom Field – A previously created distribution custom field.
  • Choices for Moment Mol Wt that the software uses for calculation:
  • Slice MW – The molecular weight of the peak slice.
  • Distribution Custom Field – A previously created distribution custom field.

Minimum area (µV · sec) (GPC only)

A minimum area the software uses to exclude distribution slices with area values less than the entered minimum when calculating molecular weight averages. Minimum Area is used with the value in the Moment Area field.

Range: Any whole number equal to or greater than 0.0 up to a maximum of 10^12

Default: Blank

Note: The Minimum Area value is used only in GPC data processing. The table field is ignored in GPCV processing.

CConst1 through CConst7

Up to seven constants to calculate custom distribution field values and custom GPC broad peak values. Enter constant values (a floating-point number) in the CConst1 to CConst7 fields. Leave these fields blank if you do not need constants in a custom database result formula.

When you use CConst1 to CConst7, set up your required custom formula before you define components in your processing method (as you would with the CCompRef fields). CConst1 to Cconst7 are included in the Edit Custom Field Formula dialog box (in Configuration Manager).
