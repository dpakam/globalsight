Processing Method tab

Use this tab to include or remove processing method information from the report. By default, items are included in the Processing Method report group, except Differences from Previous Version information.

Items to include in the report

You can include the following report items:

  • Method Information – Contents of the associated processing method.
  • Event Table (Integration/Detection Events) – Integration and detection event settings used in the associated processing method (in the Integration Event table).
  • Component Table –Component table settings used in the associated processing method.
  • Group Tables – Contents of the Timed and/or Named Group table for the associated processing method.
  • Default Value Tables – Default amount or concentration values used for the associated processing method.
  • System Suitability Limits Table – Contents of the Suitability Components table and Suitability Limit table for the associated processing method.
  • Detector Noise and Drift – Detector noise and drift parameters specified for the chromatogram in the processing method.
  • Smoothing and Time Offset – Smoothing and time offset parameters specified for the concentration, viscometer, and/or light scattering detector used for the associated processing method.

Method revision information

Note: To report on this information, Full Audit Trail must be enabled for the project.

You can include the following method revision information:

  • Revision History Details – Displays a description of the change(s) made in the Revision Description field whenever you modified the processing method (if Audit Trail Support is activated).
  • Revision Summary Information – Adds a table to the report that includes this information about the method: name, type, comments, date, modified user, locked status, method ID, and method version.
  • Differences from Previous Version – When reporting using results, you can obtain a report of the method differences between the method version used to collect or process the data and the method version before it.

Select this check box to activate these option buttons and indicate the type of differences to include in the report:

  • Print Differences Only – Includes a table in the report that only includes information for fields that differ between the two methods.
  • Print Entire Methods – Includes a table that lists the entire contents of each method with differences highlighted, allowing you to see what fields changed between the two versions.
