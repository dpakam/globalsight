Noise and Drift tab

Use this tab to report detector noise and drift results for each chromatogram. Specify a section of the chromatogram over which you want the software to calculate these results. The software calculates total noise/drift values based on the chromatogram time window (in minutes). To calculate averaged noise and drift values, specify a segment width (in seconds) and the software divides the data into segments and averages the values for each segment.

Requirement: The segment used to calculate the noise interval must include at least 30 data points. For example, when collecting data at 1 Hz, select an interval of at least 30 seconds. This prevents the artificially low noise value that would be produced if the linear regression was performed on a small number of points.  

After you process your data, the software displays the noise and drift results in the Results window, the Chromatogram Result table. The noise and drift chromatogram results include:

  • Total Values – Detector Noise, Peak to Peak Noise, and Detector Drift
  • Average Values – Average Detector Noise, Average Peak to Peak Noise, and Average Detector Drift

Calculate detector noise and drift

Enables/disables the calculation of detector noise and drift for each chromatogram as part of your processing method.

Default: Disabled

Start time (min.), stop time (min.)

The chromatogram time window (in minutes) that the software uses to calculate total and average noise and drift values.

Range: 0.000 to 655.000 min

Default: Blank

Requirements:

  • You must enter a value in both fields.
  • The Start Time must be less than the Stop Time.
  • The difference between the two times must be equal to or greater than the specified segment width.
  • The interval between the two times must contain at least 60 data points.

Segment width (sec)

Segment width (in seconds) of the chromatogram the software uses as the basis for calculating average noise and drift values.

Range: 10 to 300 sec

Default: 60 sec

Requirement: The segment width must contain at least 30 data points.
