Use this tab to select the type of calculations performed during System Suitability testing. The software uses System Suitability parameter values to calculate selectivity and capacity factor, and baseline noise and drift.

Parameter

Description

Calculate suitability results check box

Select this check box to include System Suitability testing for the processing method and calculate system suitability results and limit checking (flagging).

Default: Cleared

Calculate suitability results for unknown peaks check box

Select this check box to calculate System Suitability results for unknown peaks while also calculating System Suitability results for named peaks.

Default: Selected

System and separation efficiency parameters

Measures the separation efficiency of your system.

Parameter

Description

Void volume time (min)

Void volume time of your system that is the elution time of an unretained compound/peak under standard test conditions. Consult your column care and use manual for the test conditions. Entering the void volume time (in minutes) allows you to view results for capacity factor and selectivity.

Value range: 0.001 to 655.000 minutes

Default: Blank

Pharmacopoeia options

Calculates System Suitability results, such as selectivity and resolution, according to the formulas in these Pharmacopoeias:

  • US Pharmacopoeia (USP)
  • European Pharmacopoeia (EP)
  • Japanese Pharmacopoeia (JP).
  • All - Calculates System Suitability results for all Pharmacopoeia options, including results not specified by any of the options.

Tangent Percent

Peak height used to draw the tangents for the USP Tangent plate count and resolution calculations. When you select the US Pharmacopoeia option, Tangent Percent for USP Plate Count is set to 61 and cannot be changed.

If you select All, you can change the Tangent Percent value by entering the percent of peak height used to draw the tangents. Tangent percent determines the tangent peak width result reported in the Results window. This value is calculated only if the peak widths at the percent entered at ±5% can be calculated.

Tangent Plate Count value range:: 10 to 90%

Tangent Plate Count default: 61%

Tangent Resolution default:50%

Calculate USP, EP, and JP s/n

The s/n suitability peak fields use the signal-to-noise (s/n) formula according to the selected Pharmacopoeias.

USP s/n formula:

2 x height/(noise/scaling)

EP s/n and JP s/n formulas:

2 x (height - (0.5 x noise/scaling))/(noise/scaling)

Use noise centered on peak region in blank value

The noise value that Empower uses to calculate EP s/n according to the "Use noise centered on peak region in blank injection" check box setting. Selecting this check box enables the Half Height Multiplier fields that each pharmacopoeia uses to calculate the noise value.

When cleared, the Half Height Multiplier fields are unavailable.

Noise value for s/n

Select the type of noise that is to be used in the signal to noise (s/n) calculation.

Requirement: You must select the Calculate detector noise and drift check box in the Noise and Drift tab of the processing method and specify a time window during which noise will be calculated (start and stop time in minutes).

To specify the type of noise value to use in the noise (s/n) calculation, select one of the noise values in this field. If you do not select a value, the noise value for s/n is not calculated.

Baseline noise and drift measurement parameters

Empower software calculates noise and drift from a section of baseline within the baseline start and end times, as set in the Baseline Noise and Drift Measurements area of the Suitability tab. For valid noise calculations, the baseline interval must be free of all peaks.

Parameter

Description

Maximum allowable noise

Maximum allowable noise value. To flag noise values that exceed a limit, enter the maximum allowable noise value in the associated detector unit’s value (for example, millivolts).

Value range: 0.00 to 999.99 mV

Default: Blank

Maximum allowable drift

Maximum allowable drift value. To flag drift values that exceed a limit, enter the maximum allowable drift value in the associated detector unit's value (for example, millivolts).

Entries: 0.00 to 999.99 mV

Default: Blank

% of run time over which to average

Percentage of the run time over which to average data points. Empower software uses this value to calculate an average time, where % of Run Time Over Which to Average times the Total Run Time equals the Average Time. The software adds the Average Time to the Baseline Start Time and subtracts it from the Baseline End Time to determine the two averaged regions. Averaging occurs over averaged regions only.

Entries: 0.1 to 50.0%

Default: Blank

Baseline noise minimum

Select the minimum region for calculating baseline noise: 30 seconds (default) or 30 data points.

When the Baseline Noise Minimum is set to 30 points, if the number of data points in the baseline region is less than 30, baseline noise and drift are not calculated and an S37 processing code is added to the result.

When the Baseline Noise Minimum is set to 30 seconds:

  • If the baseline region is less than 30 seconds, baseline noise is not calculated and an S02 processing code is added to the result.
  • If the baseline region is less than 1 minute, baseline noise is not calculated, and an S01 processing code is added to the result.

Requirement: If the Baseline Noise Minimum is set to 30 seconds, the Baseline Start and Baseline End Times must be set at least 30 seconds apart.

Baseline start time (min)

Start time for drift and noise calculations. For drift calculations, the system takes a millivolt reading at the Baseline End Time, then subtracts from this reading the Baseline Start Time reading to determine drift. For noise calculations, the system calculates noise in the region of the baseline specified by the Baseline Start and End Times and the % of Run Time to Average parameter.

Default: Blank – the software uses a Baseline Start time of 0.00 minutes

Rule: The baseline interval must be free of all peaks for valid noise calculations.

Baseline end time (min)

End time for drift and noise calculations. For drift calculations, the system takes a millivolt reading at the Baseline End Time, then subtracts from this reading the Baseline Start Time reading to determine drift. For noise calculations, the system calculates noise in the region of the baseline specified by the % of Run Time Over Which to Average parameter and the Baseline Start and End Times.

Default: Blank – the software uses a Baseline End time equal to the run time.
