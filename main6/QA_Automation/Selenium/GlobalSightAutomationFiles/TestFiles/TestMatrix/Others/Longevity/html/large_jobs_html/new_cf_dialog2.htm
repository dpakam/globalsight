Data entry is required check box

Specifies that an entry is required in the custom field during sample loading (in Run Samples, the Sample Set Method Editor, or the Alter Sample window) or when you import data.

If an entry is required, you cannot proceed with sample loading until you enter a value for the field.

Default value

Enter a default value for a custom field. This default value appears in the custom field if a user entry is not required. If the data type is Bool or Enum, the default value appears at the top of the Translation column in the Translation Definition table.

Note: For Bool and Enum data types, you can enter a default value (for example, "Not Set" or leave the default blank). If you leave both the default value and the first row of the Translation Definition table blank, you can enter information for the row at run time in the Sample Method editor, the Run Samples sample table, or the Alter Sample window. This is for a sample type custom field only.

Search order

If you chose a Calculated data source (see above) and specified an intersample label, select in what order you want the database searched (Search results depend on both the combination of an intersample label used and the search order):

Option

Function

Result set first

An intersample search is performed on the current result set first. If a result which matches the current intersample label is found, the search stops (if it is not a summary search). If no result is found and the sample label is specified in an intersample label, the search continues outside the current result set to the current project. If the intersample calculation is a summary type (for example, SUM), the search continues both inside and outside of the current result set. If the label is not specified, the search ends with the current result set.  

Result set only

Intersample calculations are performed only within the current result set. If the current result is not part of a result set, the custom field will not produce a value.

Outside first

An intersample search is performed using results outside of the current result set first. If an intersample label specifies a sample label, a search is performed on all results existing in the current project first. If no result is found, the search is performed on the current result set (if there is a current result set available). If the intersample calculation is a summary type (for example, SUM), the search continues both inside and outside of the current result set. If there is no label specified in an intersample label, the search is performed on the current result set only (if available).

Outside only

Intersample calculations are performed using results from outside of the current result set. If an intersample label does not specify a sample label, a search criteria conflict occurs and the custom field will not produce a value. Outside Only has no effect in this case.

All or nothing check box

Select this check box if you do not want to generate a value on subtractions or additions unless all values are present.

Note: If you are using a summary function, All or Nothing has no effect.

Us as

If you specified a data type as either Enum or Bool in the Data and Type Selection page, select how you want to interpret or use the item you select or selected by a formula.

List selection

Description

Position

Uses the position (index) of the selected item in the Translation definition table. It starts at 0. The first item of the translation definition table is at position 0. The second one is at position 1 and etc. When Use As of a custom field is set to Position, the selected item gets translated to a position. Specifying this kind of custom field in a subsequent formula enables the use of position based calculations.

Text

Uses the selected item as a text string. This choice is useful when you need a comparison based on the selected item. The item selected is treated as text. For example, a Peak type custom field’s formula, SUM(Area*EQ(ThisCustom, “too small”)), would give you the summation of Area values of those peaks each of which ThisCustom value is translated to “too small”. ThisCustom may be an ENUM type custom field, one of which translation is “too small”.

Value

Interprets the selected item as a value. Items entered to the translation definition table must be a number. The field values generated by a custom field of this type can be used in a subsequent formula.

Note: You can choose a value from the Translation definition table as the result of the calculation of the formula described in the field, or picking it from the list.

Field

Items listed in the Enum Definition table are interpreted as either an Empower field name, a formula, or as strings. A custom field of this type can be used in a formula as long as it is specified inside of a Boolean function. For example, EQ(ThisCustom,”too small”).

Tips:

  • Field is available only if the Data Source is set to Calculated. A value generated by this custom field is stored as a string. That is the reason its usage is limited inside of a Boolean function.
  • When you select Field, you can translate the value of the custom field using the Translation Definition Table page.

Sample type

Provides the ability to calculate a custom field value selectively. Choose the Sample Type that you want to calculate on. If a formula references a field from another result, the sample from which the result is generated must match this type for the field value to be used.

Tip: Sample Type is available only if the Data Source is set to Calculated.

Peak type

Provides the ability to calculate a custom field value selectively based on the current peak type. Choose the Peak Type that you want to calculate on. If a formula references a field from another peak, its type must match this type for the field value to be used.

Tip: Peak Type is available only if the Data Source is set to Calculat

Missing peak check box

Select this option to perform calculations on missing peaks.
