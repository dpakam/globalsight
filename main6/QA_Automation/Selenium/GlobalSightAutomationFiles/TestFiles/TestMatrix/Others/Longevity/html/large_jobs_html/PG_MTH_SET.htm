Method Set tab

Use this tab to include or remove method set information. By default, all items are included in the Method Set report group, except Differences from Previous Version information.

Note: You will only obtain this information when reporting on the method itself. This information is not reported for data, results, etc.

Items to include in the report

You can include the following information in the report:

  • Method Information – Contents of the associated method set.
  • Method Set Table – Contents of the Channel table (the selected channels, processing methods, and report methods) in the method set.
  • Derived Channel Information – Contents of the derived channels used in the selected method set.
  • GPCV Channel Sets – Contents of the GPCV channel set used in the method set (GPCV option).

Method revision information

Note: To report on this information, Full Audit Trail must be enabled for the project.

You can include the following method revision information in the report:

  • Revision History Details – Displays a description of the change(s) made in the Revision Description field whenever you modified the method set (assuming the Audit Trail Support feature is activated).
  • Revision Summary Information – Adds a table to the report that includes this information about the method: name, type, comments, date, modified user, locked status, method ID, and method version.
  • Differences from Previous Version – Activates these option buttons to indicate the type of differences to include in the report:
  • Print Differences Only – Includes a table in the report that only includes information for fields that differ between the two methods.
  • Print Entire Methods – Includes a table that lists the entire contents of each method with differences highlighted, allowing you to see what fields changed between the two versions.
