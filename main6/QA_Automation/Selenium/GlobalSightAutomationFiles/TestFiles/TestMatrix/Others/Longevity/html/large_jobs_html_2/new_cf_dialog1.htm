New Custom Field wizard – Data and Type Selection page

Use this page to specify the field type and data type of the custom field.

Field type

Description

Sample (default)

Specifies a custom sample parameter (for example, pH or column type) or describes the sample (for example, lot number, analyst name, or patient ID). All data types are available for use with the Sample field type.

Result

Custom calculation of a result-level parameter (such as the total area of all peaks with a %Area less than 2% = Sum(Area*LTE(%Area,2.0)). All data types are available for use with the Result field type. You can use Real, Bool, and Enum data types with Calculated and External data sources. You can use Integer, Text, and Date data types only with an External data source.

Peak

Custom calculation of a peak-level parameter (including an interpeak calculation capability). All data types are available for use with the Peak field type. You can use Real, Bool, and Enum data types with Calculated and External data sources. You can use Integer, Text, and Date data types only with an External data source.

Sample Set

Sample set parameters (for example, column type, column lot number, plant location, and batch lot number). All data types are available for use with the Sample Set field type, but you cannot create custom calculations.

Component

Identifier that pertains to a particular peak (for example, label claim). Only the Real data type is available for use with the Component field type.

Distribution

Custom calculation of a distribution slice-level parameter for GPC/V data. Real, Bool, and Enum data types are available for use with the Distribution field type.

 

Data type

Description

Integer (0)

Whole number entry (without any fractional parts).

Real (0.0) (default)

Floating-point number entry (can contain a fraction).

Text

Alphanumeric entry.

Date

Date entry, either entered by the user or from an external source (for example, a peak field).

Tip: If your system policy is set to use long date formats and you create a custom field using a data type of date, a message informs you to enter a valid date/time. Ensure that you ensure the long date format "25 March 2012" "8:30:00" or "March 25, 2012 8:30:00".

Bool

For a Calculated Data Source, specifies an entry resulting from a Boolean logic statement (for example, a statement using the functions GT (greater than), LT (less than), GTE (greater than or equal to), and so on). For a Keyboard or External Data Source, you can enter two strings, which represent a True or False condition.

Enum

For a Calculated Data Source, specifies an enumerated entry that is used with Boolean functions. An enumerated entry is a series of Boolean statements separated by commas (,), any one of which can be true. The first true statement determines the value of the custom field. For a Keyboard or External Data Source, you can enter up to 999 definition strings.
